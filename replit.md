# Bougie Bams Website V2

A Mahjong lifestyle + events + community website for BougieBams — a Chicago-based brand bringing elevated Mahjong culture to modern players.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/db run seed` — seed the database with sample events, products, and blog posts

## Required Environment Variables

| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | Yes | Postgres connection string (auto-set by Replit DB) |
| `SESSION_SECRET` | Yes | Random secret for Express session cookie signing |
| `REPL_ID` | Yes | Auto-set by Replit; used for OIDC auth |
| `ADMIN_USER_IDS` | Yes for admin | Comma-separated list of Replit user IDs allowed to access `/admin` routes. **Set this to enable the admin panel.** Find your Replit user ID by calling `GET /api/me` after signing in — the response includes your `id`. Example: `user_abc123,user_def456` |
| `STRIPE_SECRET_KEY` | Optional | Stripe secret key for product checkout (sk_live_... or sk_test_...) |
| `STRIPE_WEBHOOK_SECRET` | Optional | Stripe webhook signing secret for payment confirmation |
| `STRIPE_SUCCESS_URL` | Optional | URL Stripe redirects to after successful checkout |
| `STRIPE_CANCEL_URL` | Optional | URL Stripe redirects to after cancelled checkout |

## Setting Up Admin Access

1. Sign in to the app (click "Sign In" in the nav)
2. After signing in, visit `/api/me` in your browser to get your user ID
3. In Replit, go to the Secrets panel and set `ADMIN_USER_IDS` to your user ID
4. Restart the API server workflow
5. Navigate to `/admin` in the app — you now have full admin access

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React 19 + Vite + wouter (routing) + Tailwind CSS v4
- API: Express 5
- DB: PostgreSQL + Drizzle ORM
- Auth: Replit OIDC (OpenID Connect)
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Payments: Stripe
- Build: esbuild (CJS bundle)

## Where Things Live

- `artifacts/bougiebams/` — React frontend (12 pages)
- `artifacts/api-server/` — Express API server
- `lib/api-spec/openapi.yaml` — single source of truth for API contract
- `lib/api-client-react/` — generated React Query hooks (do not edit directly)
- `lib/api-zod/` — generated Zod validation schemas (do not edit directly)
- `lib/db/src/schema/` — Drizzle table definitions

## Architecture Decisions

- OpenAPI spec is the source of truth; frontend hooks + Zod schemas are generated by Orval
- `requireAdmin` is secure-by-default: denies all unless `ADMIN_USER_IDS` is set and includes the requesting user
- `POST /registrations` only allows free events (priceCents null/0); paid events must use `POST /registrations/checkout` via Stripe
- `@tanstack/react-query` is a peerDependency in api-client-react (not a dependency) to prevent duplicate React Query instances; it's also a devDependency for typecheck

## Gotchas

- After any API server code change, rebuild: `pnpm --filter @workspace/api-server run build`, then restart the "API Server" workflow
- After any OpenAPI spec change, run codegen: `pnpm --filter @workspace/api-spec run codegen`
- Before api-server typecheck, build composite TS libs: `npx tsc -b lib/db/tsconfig.json lib/api-zod/tsconfig.json`
- The seed script is idempotent — it skips tables that already have rows
